# Growth Hacking
Growth Hacking은 IT 산업의 성장에 따라 새롭게 도래한 성장 방정식이다.

- 새로운 마켓의 출현
- 서비스의 지속적인 업데이트가 가능함
- 클라우드 환경으로 인해 작은 팀으로도 지속가능한 서비스 개발 가능
- 고객에 대한 데이터 수집이 매우 유용한 다양한 도구들을 활용
- 온라인 마케팅 채널의 활성화 및 고도화

## 기업의 성장과 Growth Hacking
일반적으로 모든 기업은 J Curve 를 달성하고 싶어한다. 


### 기본 성장 방정식
- 더 많은 사람을 모으고, 
- 고객으로 만들고, 
- 매출을 발생


### 기본 성장 방정식에서 User Lifecycle 로 인식변화 
그로스 해킹은 중요한 3가지 단계뿐만 그 중간에 있는 모든 과정들을 데이터를 기반으로 세부적으로 분석하여 그 결과를 가지고 지속적으로 실험하고, 실험된 결과를 바탕으로 서비스가 지속적으로 성장할 수 있도록 돕는데 목표가 있다.

- 더 많은 사람을 모으고, 
- (그로스 해킹해서는 세부적으로 더 많은 단계로 쪼갬)
- 고객으로 만들고, 
- (그로스 해킹해서는 세부적으로 더 많은 단계로 쪼갬)
- 매출을 발생

## Growth Hacking 개념

### 전환율(Conversion)
- '목표에 접근하는 사용자 대비 실제 목표를 달성한 사용자'
- 목표와 달성을 제대로 정의하는 것이 CVR 측정의 첫번째 단계

이 공식에서 사용할 수 있는 분모와 분자를 잘못 정의할 경우 전환율이 낮거나 높은 결과를 보게 되고, 이 데이터가 지속적으로 쌓였을 때, 지속적으로 잘못된 의사결정할 확률이 높아지므로 전환율을 구하는 분모와 분자를 정확하게 정의하고 있는지, 측정하고 있는지를 반드시 짚어야 한다. 

### 퍼널(Funnel)
목표를 달성하기 위해 반드시 거쳐야 하는 단계를 분석하여 단계별로 서로 다른 전략을 사용하여 개선함으로써 목표하는 전환율을 달성

> 목표를 달성하는 과정을 퍼널이라고 함


- 각각의 전환율을 높이는 법
- 실제 상품이 고객에게 얼마나 유용한지, 잘 추천되었는지를...! 
- 전체 부서가 퍼널을 잘 살펴봐야 한다.

## 고객 생명주기에 따른 Growth Hacking 기법
어떤 지표에 대해 궁금해야 하고, 어떤 지표를 측정해야 하는가?

### Acquisition
Acquisition 은 마케팅 자체에 밀접한 관련이 있다. 

#### 효과적인 채널은 무엇일까?
- 비용? 효율적인 비용이란? CPA?
- 볼륨? 채널의 볼륨이란?

#### 효율적인 Landing Page는 무엇일까? 
- 신규? 어떤 단계로 Landing?
- 채널별로?
- A/B 테스팅
- 모바일 웹으로(설치 유도)?
- 스토어로 곧바로?

### Activation
Activation 에서 마케팅, 세일즈, 상품이 상호작용하지 못하면 Revenue로 이어지기 어렵다.

- 회원가입?
  - 반드시 필요한가? 비회원 주문?
  - 소셜 로그인
- 홈 화면
  - 추천? (랭킹)
  - 검색?
- 첫 번째 고객의 전환(Goal)이란?
- 세그먼트를 위한 장치는?
  - 행동 정보?
  - 추가 정보 입력?

### Retention
- 유입 경로
  - Direct? Engagement? Retargeting
  - Email / SMS / Push
- 세그먼테이션
  - 행복 정보/주문 정보/DMP(타 사이트 고객 정보)
- 다양한 이벤트?


### Revenue
- ROAS: 광고비 집행 대비 구매로 이어진 세일즈 단가
  - 실패율 반영?
- 결제까지의 경로
  - 최소화할 수 없는가? (장바구니 필요?)
  - CTA를 높이기 위한 부가 정보들
  - 다양한 결제 수단
- 도움 제공
  - Chatbot, 전화, FAQ
- Exit poll 또는 만족도 조사
  - 만족도 👉  App Review, Referral

### Referral
- 혜택
  - No point(바이럴)
  - 쿠폰 / 할인
- 전달 정보
  - 상품 정보? 친구 정보?
- 전달 방법
  - FB 공유 / 쿠폰 공유 / SMS / Kakao / Email
- CPA 측정 
  - LTV 관점에서 의미있는가?


## Growth Hacking 사례 분석

### Case Study - 전환율 개선
사용자가 빠르게 원하는 정보를 찾을 수 있도록 하자 👉  '검색'기능의 추가 및 강화

> Data에 기반한 Total Soccer!

- 검색 기능을 강화하지 않은 이유 
  - '우리가 고객에 대해서 더 잘 알고 있다' 라는 확신!
  - **큐레이션** 중심이므로 랭킹 적용시 서비스 성공 보장 못함
  - 더 많은 호텔/팬션 정보를 확인하지 않을지도 모른다는 불안감
- 목표
  - 고객에 대해서 더 잘 이해하자
  - 랭킹의 호율을 측정해보고 검색부터 적용하자
  - 비즈니스 모델을 랭킹에 반영하자
  - 빠른 구매 전환을 유도해보자

### Case Study - 개선 아이템 도출
- 검색을 통한 호텔 목록 진입시 구매 전환율 높음
  - 하지만 잠식효과 유의 => 전체 CVR이 높아지는가? 
- 호텔 목록 진입의 15 ~ 20%가 검색화면 진입
  - 구매 전환율이 보장되면 검색 화면 노출을 강화하기 위해 UX 개선

- 지역 키워드 검색 비율 높음
  - 고객은 지역 선택도 귀찮아 함. 지역
- 호스텔, 펜션, 레지던스, 리조트, 풀빌라, 모텔과 같은
- 드물지만 드마리스와 고객 걱정을고메 검색 의도 있음 => 고메 검색 기능 제공 예정
