## 리액트를 들어가기 앞서서

단순히 웹 페이지를 통해 정보만을 보여주는 게 아니라 사용자와의 상호작용이 자주 발생하고, 그에 따라 UI를 표현해야 하므로 웹은 RIA(Rich Internet Application) 으로 발전했다..!
백엔드 혼자 짊어지던 책임을 나누어지기 위해서 프론트 엔드 기술이 발전..!했다. 

숙련된 JavaScript 개발자라면, 코드를 최대한 깔끔하게 정리하여 
유지보수가 쉽도록 준비해놓을 수 있겠지만
숙련되지 않은 JavaScript 개발자에게는 어려운 숙제가 될 것입니다. 

규칙이 다양해지고, 관리하기 힘든 상황에서 웹 애플리케이션의 규모가 커지면, 대부분 DOM 을 직접 건드리면서 작업을 하는 경우가 생기고 이러면 코드가 난잡해지기 쉽다.



리액트 컴포넌트는 함수형태로 작성 할 수도 있고 클래스형태로도 작성 할 수 있다. 
리액트 컴포넌트에서는 XML 형식의 값을 반환해줄 수 있는데 이를 JSX 라고 부른다.
